Explication des différents répertoires :

Les sources de ce guide sont écrites en Markdown et sont situées dans le dossier markdown.

La concaténation des fichiers .md et la version html brute sont situés dans le dossier output

La version html + Bootstrap est situé dans le dossier build.

Pre-compilation

Pour précompiler, j'utilise Gruntjs. Différentes tâches sont disponibles :

  • default : tâche par défaut. Ne fais que concaténer les fichiers .md du répertoire markdown en un seul fichier : output/build.md

  • layout : concatène, et sort une version bootstrap + html

  • concatOnly : même tâche que default

  • noLayout : concatène, et sort une version brute en .html

  • De plus, ce lisez-moi peut être compilé grâce à la tâche readme

Pour éxécuter l'une de ces tâche écrire dans le terminal : 

grunt <nom de tâche>

Exemple avec la tâche layout : 

grunt layout

Pre-processing

To pre-process, i use Gruntjs. Several tasks are available :

In order to execute one of these tasks, type in a console :

Example with layout task :
